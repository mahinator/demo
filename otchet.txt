ВВЕДЕНИЕ
-------------------------------------------------------------------------------
В рамках демонстрационного экзамена разработана информационная система для ООО "МирИгрушек" (магазин игрушек). Реализованы модули создания базы данных, авторизации по ролям, управления товарами и заказами, а также развертывания на удаленном веб-сервере.

МОДУЛЬ 1. РАЗРАБОТКА И НАСТРОЙКА БАЗЫ ДАННЫХ
-------------------------------------------------------------------------------
1. Спроектирована реляционная база данных "demo" в 3НФ. Созданы 10 таблиц:
   - roles, users (пользователи и роли)
   - categories, manufacturers, suppliers (справочники товаров)
   - products (товары)
   - pickup_points (36 пунктов выдачи)
   - order_statuses, orders, order_items (заказы и их состав)
2. Все таблицы связаны внешними ключами с ограничениями ссылочной целостности.
3. Выполнен импорт исходных данных из файлов ресурсов (.xlsx).
4. Разработан SQL-скрипт script.sql для развертывания структуры и данных.

МОДУЛЬ 2. РАЗРАБОТКА АЛГОРИТМА И СОЗДАНИЕ ПРИЛОЖЕНИЯ
-------------------------------------------------------------------------------
1. Разработана и оформлена блок-схема алгоритма работы приложения по ГОСТ 19.701-90.
2. Создан единый дизайн согласно руководству по стилю (шрифт Arial, цвета: #FFFFFF, #F5DEB3, #DEB887).
3. Добавлена иконка icon.png рядом с логотипом ООО "МирИгрушек" в окне авторизации.
4. Разработана страница входа (index.php) с разделением прав по ролям:
   - Администратор (полный доступ к товарам и заказам)
   - Менеджер (фильтры, сортировка, просмотр заказов)
   - Клиент (просмотр товаров без поиска/фильтров)
   - Гость (просмотр товаров без поиска/фильтров, вход по кнопке)
5. Реализован вывод списка товаров в виде прямоугольных плашек (макет: фото слева, описание по центру, скидка крупно справа).
6. Настроена автоматическая подсветка товаров:
   - Скидка > 17% (фоновый цвет #FFDEAD).
   - Товар отсутствует на складе (голубой фон #B0E0E6).
   - При наличии скидки старая цена перечеркнута красным, новая выведена черным.

МОДУЛЬ 3. ПОСЛЕДОВАТЕЛЬНЫЙ UI. ФОРМЫ ТОВАРОВ. УДАЛЕНИЕ.
-------------------------------------------------------------------------------
1. Обеспечен сквозной последовательный интерфейс (кнопки "Назад" на всех формах, кнопка "Выйти").
2. Реализованы поиск в реальном времени (oninput), фильтрация по поставщику и сортировка по цене/остатку. Все фильтры работают совместно, фокус в поиске сохраняется.
3. Разработаны формы добавления и редактирования товаров (для Администратора):
   - Значения подгружаются из СУБД (выпадающие списки).
   - ID товара при добавлении рассчитывается автоматически (+1 в БД), а при редактировании открыт только для чтения.
   - Валидация цен, остатков и скидок на неотрицательность.
4. Загрузка фото с проверкой разрешения (не более 300x200 px) и автоудалением старого файла при замене.
5. Запрещено удаление товаров, присутствующих в заказах.
6. Разработаны простые Toast-уведомления (на сером фоне с эмодзи: ✅, ❌, ⚠️, ℹ️) для вывода ошибок и предупреждений.
7. Реализовано модальное окно подтверждения удаления товаров в простом дизайне.
